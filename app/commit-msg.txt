fix: drop country flags from language picker, sync html lang

Two language-selection fixes:
- Removed the country-flag emojis from the language picker. A country
  flag doesn't represent a language (Russian isn't only Russia, English
  isn't only the UK), so only the language names remain.
- Sync <html lang> with the active language on web. The web build
  advertised the default lang (en) even when the UI was Russian, so
  Chrome offered to translate the page. Setting documentElement.lang on
  init and on change stops that and improves screen-reader accuracy.
