fix: set a localized document title on web

document.title was empty, leaving Chrome without a strong language
signal, so it offered to translate the page even though <html lang> is
ru and the visible UI is Russian (likely triggered by Latin text in the
data — emails, names). syncHtmlLang now also sets a localized title
("Моё собрание" / "My Congregation" / "Meine Versammlung") alongside the
lang attribute, which names the browser tab properly and gives Chrome a
clear language signal.
